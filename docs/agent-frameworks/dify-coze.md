# Dify 与 Coze 低代码 Agent 平台

并非所有 Agent 应用都需要从代码层面构建。对于产品经理、运营人员、业务专家等非技术背景的用户，低代码 Agent 平台提供了另一种路径——通过可视化界面定义 Agent 的行为、编排工作流、接入外部工具，无需编写代码即可搭建可运行的 Agent 应用。Dify 和 Coze 是这一领域的两个代表性平台。

## Dify：开源的 AI 应用开发平台

Dify 是一个开源的 LLM 应用开发平台，定位在"低代码但保留代码扩展能力"的中间地带。

### 可视化编排

Dify 的核心是**工作流（Workflow）编辑器**——一个基于节点的可视化编排画布：

- **LLM 节点**：调用大模型进行推理，可配置模型选择、系统提示词、温度等参数。
- **知识检索节点**：从已索引的知识库中检索相关文档片段，作为 LLM 的上下文输入。
- **代码节点**：执行 Python 或 JavaScript 代码片段，处理数据转换、格式化等逻辑。
- **条件分支节点**：根据上游节点的输出进行条件判断，将流程导向不同分支。
- **工具节点**：调用预置或自定义的外部工具/API。
- **变量聚合节点**：合并多个分支的输出，统一传递给下游节点。

工作流支持两种运行模式：**Chatflow**（对话型，多轮交互）和 **Workflow**（批处理型，单次输入输出）。开发者可以在画布上拖拽节点、连线、配置参数，实时预览执行结果，无需关心底层代码实现。

### 知识库与 RAG

Dify 内置了完整的 RAG（检索增强生成）能力：

- 支持多种文档格式上传（PDF、Word、TXT、Markdown 等）
- 自动分段与索引，可选多种 Embedding 模型
- 多种检索策略（向量检索、全文检索、混合检索）
- 检索结果可直接在工作流中作为 LLM 节点的上下文使用

这使得知识型 Agent 的搭建变得极为简便——上传文档、配置检索节点、设置提示词，即可上线一个具备领域知识回答能力的 Agent。

### 插件生态与 API

Dify 的工具生态通过**API 工具**扩展——用户可以自定义 HTTP API 调用作为工具节点，定义请求格式和响应解析规则。平台也预置了搜索引擎、天气查询等常用工具。更关键的是，Dify 提供 RESTful API，允许外部应用调用 Dify 上创建的 Agent，实现了"低代码创建、代码级调用"的组合模式。

### 开源优势

Dify 的开源特性意味着：可以私有化部署、自定义前端界面、修改后端逻辑、对接内部系统。对于有合规要求或需要深度定制的企业场景，这是一个重要优势。

## Coze：字节跳动的 Agent 创作平台

Coze（扣子）是字节跳动推出的 Agent 创作平台，在国内市场有较大的用户基础，定位更偏向"零代码"。

### 可视化编排

Coze 的工作流编辑器与 Dify 类似，基于节点拖拽编排，但更强调**对话型 Agent** 的构建：

- **Bot 模式**：定义一个对话型 Agent，配置人设、开场白、知识库、插件，用户与 Bot 直接对话交互。
- **Workflow 模式**：定义后台处理流程，可被 Bot 调用或在独立场景中使用。
- **触发器**：支持定时触发、事件触发，让 Agent 可以主动推送而非被动响应。

### 插件生态

Coze 的插件生态是其差异化优势：

- **官方插件库**：预置了搜索、新闻、地图、图片生成、数据分析等数十种插件，大部分可以直接启用无需配置。
- **插件创建工具**：用户可以基于现有 API 创建自定义插件，Coze 提供了 API 调试和参数映射工具。
- **插件市场**：社区共享的插件生态，用户可以搜索和使用他人发布的插件。
- **多模态支持**：图片生成、语音合成、视频处理等插件让 Agent 的输出不局限于文本。

Coze 的插件接入体验较为顺畅——大部分官方插件一键启用，参数配置简化到最低程度，降低了非技术用户的使用门槛。

### 发布与分发

Coze 的另一大特色是**多渠道发布**：

- 支持发布到飞书、微信、掘金等国内主流平台
- 支持生成独立的 Web 对话界面
- 支持通过 API 接入自定义前端

这种分发能力让 Agent 应用可以快速触达终端用户，从创建到上线的闭环时间极短。

## 适用场景与局限性

### 适用场景

- **快速原型验证**：需要快速验证 Agent 应用可行性的场景，低代码平台可以在数小时内搭建可交互原型。
- **非技术团队自主构建**：产品、运营、客服等团队需要自主创建和维护 Agent 应用，无需依赖开发团队排期。
- **标准化的知识问答应用**：基于 RAG 的领域知识问答是最适合低代码平台的场景——工作流简单、知识库内置、效果可预期。
- **内部工具与流程自动化**：企业内部的审批流程、数据查询、报告生成等标准化流程。

### 局限性

- **复杂逻辑受限**：当 Agent 需要复杂的状态管理、动态决策路径、多层嵌套调用时，可视化节点的表达能力不如代码级框架。条件分支节点能处理简单分支，但难以表达 LangGraph 状态图级别的复杂流程。
- **调试与可观测性不足**：低代码平台的执行日志通常简化为节点级别的输入输出摘要，缺少推理过程的细粒度追踪。Agent 行为异常时的排查难度高于代码级框架。
- **定制上限**：Dify 的开源性质缓解了这个问题，但 Coze 等闭源平台的定制空间有限——前端风格、交互逻辑、部署方式都受平台约束。
- **模型选择受限**：平台通常只支持接入已集成的模型提供商，无法像代码级框架那样灵活切换或混合使用多个模型。
- **数据与合规风险**：闭源平台的应用数据和用户交互数据存储在平台侧，对于有数据安全要求的企业场景需要谨慎评估。

低代码平台和代码级框架不是替代关系而是互补关系——前者加速原型和标准化应用的落地，后者承载复杂定制化需求。在 Agent 应用的发展路径上，通常从低代码快速验证开始，验证可行性后再用代码级框架重构生产版本。
